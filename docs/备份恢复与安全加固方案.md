## 1. 数据库备份与灾难恢复

### 1.1 备份策略总览

| 策略类型 | 频率 | 工具/方式 | 保留周期 | 存储位置 |
|---------|------|----------|---------|---------|
| 全量备份 | 每天 02:00 | `pg_dump` | 7 天 | 本地 + OSS/S3 |
| WAL 连续归档 | 实时 | `archive_command` | 7 天 | 本地 + OSS/S3 |
| 异地同步 | 每天 04:00 | `rclone` / 阿里云 OSS CLI | 7 天 | 阿里云 OSS / AWS S3 |

### 1.2 全量备份（pg_dump）

#### 1.2.1 Shell 脚本

```bash
#!/bin/bash
# /opt/scripts/pg_full_backup.sh

set -euo pipefail

DB_NAME="sci_agent"
DB_USER="sci_agent_admin"
BACKUP_DIR="/data/backups/postgres/full"
RETENTION_DAYS=7
TIMESTAMP=$(date +"%Y%m%d_%H%M%S")
BACKUP_FILE="${BACKUP_DIR}/${DB_NAME}_${TIMESTAMP}.dump.gz"

mkdir -p "$BACKUP_DIR"

pg_dump -U "$DB_USER" -d "$DB_NAME" \
    --format=custom \
    --compress=9 \
    --verbose \
    --file="$BACKUP_FILE" 2>&1 | tee "${BACKUP_DIR}/${DB_NAME}_${TIMESTAMP}.log"

# 校验备份文件完整性
if ! gunzip -t "$BACKUP_FILE" 2>/dev/null; then
    echo "FATAL: Backup file integrity check failed!" >&2
    exit 1
fi

# 清理超过保留天数的本地备份
find "$BACKUP_DIR" -name "${DB_NAME}_*.dump.gz" -mtime +${RETENTION_DAYS} -delete
find "$BACKUP_DIR" -name "${DB_NAME}_*.log" -mtime +${RETENTION_DAYS} -delete

echo "Full backup completed: $BACKUP_FILE"
```

#### 1.2.2 Cron 定时任务

```cron
0 2 * * * /opt/scripts/pg_full_backup.sh >> /var/log/pg_backup_cron.log 2>&1
```

### 1.3 WAL 连续归档（PITR）

#### 1.3.1 postgresql.conf 配置

```ini
# Enable WAL archiving
wal_level = replica
archive_mode = on
archive_command = 'test ! -f /data/backups/postgres/wal/%f && cp %p /data/backups/postgres/wal/%f'
archive_timeout = 60
wal_keep_size = 1024    # 1GB
```

#### 1.3.2 说明

- `archive_command` 将完成的 WAL 段实时复制到归档目录。
- 结合全量备份 + WAL 归档，可实现任意时间点的数据恢复（Point-in-Time Recovery）。
- WAL 归档目录按同样的保留策略（7 天）清理过期文件：

```bash
# /etc/cron.daily/pg_wal_cleanup
find /data/backups/postgres/wal/ -type f -mtime +7 -delete
```

### 1.5 恢复流程

| 指标 | 目标值 | 说明 |
|------|-------|------|
| **RTO** (恢复时间目标) | ≤ 30 分钟 | 从故障发生到服务恢复的时长 |
| **RPO** (恢复点目标) | ≤ 5 分钟 | 允许的最大数据丢失窗口（得益于 WAL 连续归档） |

#### 1.5.1 恢复步骤

```bash
#!/bin/bash
# 灾难恢复完整流程

# ============ 步骤 1：从 OSS/S3 拉取备份文件 ============
RESTORE_DATE="20260701"   # 指定要恢复到的全量备份日期

mkdir -p /tmp/pg_restore
rclone copy s3_backup:sci-agent-db-backups/full/ \
    --include "sci_agent_${RESTORE_DATE}*.dump.gz" \
    /tmp/pg_restore/

# 同时拉取对应日期的 WAL 归档
rclone sync s3_backup:sci-agent-db-backups/wal/ /tmp/pg_restore/wal/

# ============ 步骤 2：停止应用服务 ============
systemctl stop sci-agent-api
systemctl stop sci-agent-worker

# ============ 步骤 3：清理现有数据库（可选：先备份当前状态） ============
# pg_dump -U sci_agent_admin sci_agent > /tmp/pre_restore_dump.sql
psql -U sci_agent_admin -c "DROP DATABASE IF EXISTS sci_agent_restore;"
psql -U sci_agent_admin -c "CREATE DATABASE sci_agent_restore OWNER sci_agent_admin;"

# ============ 步骤 4：恢复全量备份 ============
pg_restore -U sci_agent_admin \
    -d sci_agent_restore \
    --verbose \
    --clean \
    --if-exists \
    /tmp/pg_restore/sci_agent_${RESTORE_DATE}_*.dump.gz

# ============ 步骤 5：（可选）PITR 恢复到指定时间点 ============
# 编辑 restore_command 配置
cat >> /var/lib/postgresql/data/postgresql.conf <<EOF
restore_command = 'cp /tmp/pg_restore/wal/%f %p'
recovery_target_time = '2026-07-01 14:30:00'
EOF

# 创建 recovery.signal 信号文件
touch /var/lib/postgresql/data/recovery.signal
pg_ctl restart

# 等待恢复完成后清理
rm -f /var/lib/postgresql/data/recovery.signal

# ============ 步骤 6：交换数据库 ============
psql -U sci_agent_admin -c "
  ALTER DATABASE sci_agent RENAME TO sci_agent_old;
  ALTER DATABASE sci_agent_restore RENAME TO sci_agent;"

# ============ 步骤 7：验证恢复数据完整性 ============
psql -U sci_agent_admin -d sci_agent -c "
  SELECT count(*) AS user_count FROM users;
  SELECT count(*) AS paper_count FROM papers;
  SELECT count(*) AS task_count FROM tasks;
  SELECT max(created_at) AS latest_record FROM audit_logs;"

# ============ 步骤 8：重启应用服务 ============
systemctl start sci-agent-api
systemctl start sci-agent-worker

# 验证 API 健康检查
curl -f http://localhost:8000/health || exit 1

echo "Recovery completed successfully!"
```

## 2. API 安全加固

### 2.1 Rate Limiting（速率限制）

#### 2.1.1 分档策略

| 用户级别 | 限制（req/min） | 限制（req/hour） | 维度 |
|---------|----------------|-----------------|------|
| 未认证 | 60 | — | IP |
| 已认证（免费） | 300 | 5000 | user_id |
| 已认证（付费） | 1000 | 20000 | user_id |

#### 2.1.2 基于 Redis Sliding Window 的实现

```python
# app/middleware/rate_limiter.py
import time
import redis.asyncio as aioredis
from fastapi import Request, HTTPException
from starlette.middleware.base import BaseHTTPMiddleware

RATE_LIMITS = {
    "anonymous":  60,    # req/min
    "authenticated_free": 300,
    "authenticated_premium": 1000,
}


class SlidingWindowRateLimiter(BaseHTTPMiddleware):
    def __init__(self, app, redis_url: str):
        super().__init__(app)
        self.redis = aioredis.from_url(redis_url, decode_responses=True)

    async def dispatch(self, request: Request, call_next):
        key = self._build_key(request)
        limit = self._get_limit(request)

        now = time.time()
        window_start = now - 60  # 1-minute sliding window

        async with self.redis.pipeline(transaction=True) as pipe:
            pipe.zremrangebyscore(key, 0, window_start)  # 移除过期记录
            pipe.zcard(key)                                # 当前窗口请求数
            pipe.zadd(key, {str(now): now})               # 添加当前请求
            pipe.expire(key, 61)                           # 设置 key 过期
            _, count, _, _ = await pipe.execute()

        if count >= limit:
            retry_after = 60
            raise HTTPException(
                status_code=429,
                detail={"error": "Too Many Requests", "retry_after": retry_after},
                headers={"Retry-After": str(retry_after)},
            )

        response = await call_next(request)
        return response

    def _build_key(self, request: Request) -> str:
        user_id = getattr(request.state, "user_id", None)
        if user_id:
            return f"ratelimit:user:{user_id}"
        client_ip = request.client.host
        return f"ratelimit:ip:{client_ip}"

    def _get_limit(self, request: Request) -> int:
        user = getattr(request.state, "user", None)
        if user is None:
            return RATE_LIMITS["anonymous"]
        tier = getattr(user, "tier", "free")
        if tier == "premium":
            return RATE_LIMITS["authenticated_premium"]
        return RATE_LIMITS["authenticated_free"]
```

#### 2.1.3 注册中间件（app/main.py）

```python

    SlidingWindowRateLimiter,
    redis_url="redis://localhost:6379/0",
)
```

### 2.3 请求体大小限制

```python
# app/main.py
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.responses import JSONResponse

MAX_BODY_SIZE = 10 * 1024 * 1024  # 10 MB


class BodySizeLimiter(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        content_length = request.headers.get("content-length")
        if content_length and int(content_length) > MAX_BODY_SIZE:
            return JSONResponse(
                status_code=413,
                content={"detail": "Request body too large. Maximum allowed size is 10 MB."},
            )
        return await call_next(request)


app.add_middleware(BodySizeLimiter)
```

> **补充**：生产环境建议同时在反向代理层（Nginx）设置 `client_max_body_size 10m;` 作为第一道防线。

### 2.4 SQL 注入防护

#### 2.4.1 原则

| 规则 | 说明 |
|------|------|
| ✅ 强制使用 SQLAlchemy ORM / 参数化查询 | 所有数据库操作走 `session.query()` / `session.execute(text(...), params)` |
| ❌ 禁止裸写拼接 SQL | 严禁 `f"SELECT * FROM users WHERE name = '{user_input}'"` |
| ✅ CR 审查 | 所有含原始 SQL 的代码必须经过 Code Review |

#### 2.4.2 正确示例

```python
# ✅ 正确：SQLAlchemy ORM
user = session.query(User).filter(User.email == user_input).first()

# ✅ 正确：参数化查询
from sqlalchemy import text
result = session.execute(
    text("SELECT * FROM papers WHERE title ILIKE :query"),
    {"query": f"%{user_input}%"},
)

# ✅ 正确：使用 insert/update 构建器
stmt = update(User).where(User.id == user_id).values(name=new_name)
session.execute(stmt)
```

#### 2.4.3 错误示例（绝对禁止）

```python
# ❌ 危险：字符串拼接
session.execute(f"SELECT * FROM users WHERE email = '{user_input}'")

# ❌ 危险：format 拼接
session.execute("SELECT * FROM users WHERE id = {}".format(user_id))
```

### 2.5 Helmet 安全响应头中间件

```python
# app/middleware/security_headers.py
from starlette.middleware.base import BaseHTTPMiddleware


class SecurityHeadersMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        response = await call_next(request)
        response.headers.update({
            "X-Content-Type-Options": "nosniff",
            "X-Frame-Options": "DENY",
            "X-XSS-Protection": "0",  # 现代浏览器已弃用，设为 0 禁用，改用 CSP
            "Referrer-Policy": "strict-origin-when-cross-origin",
            "Strict-Transport-Security": "max-age=31536000; includeSubDomains; preload",
            "Content-Security-Policy": (
                "default-src 'self'; "
                "script-src 'self'; "
                "style-src 'self' 'unsafe-inline'; "
                "img-src 'self' data: https:; "
                "connect-src 'self' https://api.sci-agent.example.com; "
                "frame-ancestors 'none'; "
                "base-uri 'self'; "
                "form-action 'self';"
            ),
            "Permissions-Policy": (
                "camera=(), "
                "microphone=(), "
                "geolocation=(), "
                "interest-cohort=()"
            ),
            "Cross-Origin-Opener-Policy": "same-origin",
            "Cross-Origin-Embedder-Policy": "require-corp",
            "Cross-Origin-Resource-Policy": "same-origin",
        })
        return response
```

### 2.6 CSRF 保护

#### 2.6.1 方案概述

采用双层防护：**SameSite Cookie + CSRF Token**。

| 层级 | 机制 | 覆盖场景 |
|------|------|---------|
| Cookie 层 | `SameSite=Strict` / `SameSite=Lax` | 浏览器自动携带 Cookie 的跨站请求 |
| Token 层 | CSRF Token（Double Submit Cookie 模式） | 对 SameSite 不完全兼容的场景兜底 |

#### 2.6.2 配置

```python
# app/main.py
from starlette.middleware.sessions import SessionMiddleware

app.add_middleware(
    SessionMiddleware,
    secret_key="YOUR_SECRET_KEY",  # 使用环境变量注入
    session_cookie="session_id",
    same_site="lax",              # SameSite 防护
    https_only=True,              # 仅 HTTPS 传输
    max_age=86400,
)
```

#### 2.6.3 CSRF Token 端点与验证

```python
# app/api/csrf.py
import secrets
from fastapi import APIRouter, Request, HTTPException

router = APIRouter(prefix="/csrf", tags=["csrf"])


@router.get("/token")
async def get_csrf_token(request: Request):
    """前端在每次 SPA 加载时获取 CSRF Token"""
    token = secrets.token_hex(32)
    request.session["csrf_token"] = token
    return {"csrf_token": token}


# app/middleware/csrf.py
async def csrf_middleware(request: Request, call_next):
    if request.method in ("GET", "HEAD", "OPTIONS", "TRACE"):
        return await call_next(request)

    # 从请求头获取 CSRF Token
    header_token = request.headers.get("X-CSRF-Token")
    session_token = request.session.get("csrf_token")

    if not header_token or not session_token:
        raise HTTPException(status_code=403, detail="CSRF token missing")

    # 常量时间比较，防时序攻击
    if not secrets.compare_digest(header_token, session_token):
        raise HTTPException(status_code=403, detail="CSRF token mismatch")

    return await call_next(request)
```

#### 2.6.4 前端配合（示例）

```javascript
// 请求拦截器中自动附加 CSRF Token

  const token = sessionStorage.getItem('csrf_token');
```

### 3.2 CI 安全扫描集成

#### 3.2.1 GitHub Actions Workflow

```yaml
# .github/workflows/security-scan.yml
name: Security Scan

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]
  schedule:
    - cron: "0 3 * * 1"   # 每周一凌晨 3:00

jobs:
  pip-audit:
    name: Python Dependency Audit
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - name: Install pip-audit
        run: pip install pip-audit

      - name: Run pip-audit
        id: audit
        run: |
          pip-audit \
            --requirement requirements.txt \
            --format json \
            --output pip-audit-report.json \
            --strict \
            || true   # 先不中断，后续按严重级别判断

      - name: Check for HIGH/CRITICAL vulnerabilities
        run: |
          python -c "
          import json
          with open('pip-audit-report.json') as f:
              data = json.load(f)
          high_or_critical = [
              v for v in data.get('dependencies', [])
              for vuln in v.get('vulns', [])
              if vuln.get('severity') in ('high', 'critical')
          ]
          if high_or_critical:
              print(f'FATAL: Found {len(high_or_critical)} HIGH/CRITICAL vulnerabilities')
              # 重新以非零退出码失败
              exit(1)
          else:
              print('No HIGH or CRITICAL vulnerabilities found.')
          "

  npm-audit:
    name: NPM Dependency Audit
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: ./frontend
    steps:
      - uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: "20"
          cache: "npm"
          cache-dependency-path: frontend/package-lock.json

      - name: Install dependencies
        run: npm ci

      - name: Run npm audit
        id: audit
        run: |
          npm audit --json > npm-audit-report.json || true

      - name: Check for HIGH/CRITICAL vulnerabilities
        run: |
          node -e "
          const data = require('./npm-audit-report.json');
          if (!data.vulnerabilities) process.exit(0);
          const highOrCritical = Object.entries(data.vulnerabilities)
            .filter(([_, v]) => v.severity === 'high' || v.severity === 'critical');
          if (highOrCritical.length > 0) {
            console.error('FATAL: Found ' + highOrCritical.length + ' HIGH/CRITICAL vulnerabilities');
            highOrCritical.forEach(([name, v]) => console.error(' - ' + name + ': ' + v.severity));
            process.exit(1);
          } else {
            console.log('No HIGH or CRITICAL vulnerabilities found.');
          }
          "

  sast:
    name: SAST (Bandit + Semgrep)
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Run Bandit (Python SAST)
        run: |
          pip install bandit
          bandit -r src/ -f json -o bandit-report.json
          bandit -r src/ --severity-level high -ll -q || exit 1

      - name: Run Semgrep
        uses: semgrep/semgrep-action@v1
        with:
          config: p/python
        continue-on-error: false
```
