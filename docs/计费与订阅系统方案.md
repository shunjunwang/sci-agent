## 1. 定价模型

### 1.1 C 端订阅

| Plan | 年费 | 核心权益 |
|---|---|---|
| **Basic** | 99 元/年 | 基础文献检索、单篇翻译、5 个项目空间 |
| **Pro** | 199 元/年 | 高级检索过滤器、批量翻译、无限项目空间、优先客服 |
| **Enterprise** | 268 元/年 | 全功能开放、团队协作（上限 10 人）、SSO 登录、API 访问 |

### 1.2 B 端私有化部署

| 部署规模 | 价格区间 | 说明 |
|---|---|---|
| 小型团队（≤30 人） | 1.8w ~ 3.6w 元 | 单机部署、基础运维支持 |
| 中型团队（30~100 人） | 3.6w ~ 7.2w 元 | 集群部署、7×12 运维、定制对接 |
| 大型机构（100 人以上） | 7.2w ~ 12w 元 | 高可用集群、7×24 运维、深度定制、培训 |

### 1.3 增值分成

- 通过 SCI-Agent 平台产生的增值服务（人工润色、期刊推荐、投稿辅导等），平台与 B 端服务商按 **3:7** 分成。
- 结算周期：自然月，次月 10 日前出账单。

## 2. 数据库设计

> 以下 4 张表追加在 SPEC.md 已有 16 张表之后，与其他表共享同一数据库。

### 2.1 表结构

#### subscriptions（订阅表）

```sql
CREATE TABLE subscriptions (
    id              BIGSERIAL PRIMARY KEY,
    user_id         BIGINT       NOT NULL,
    plan            VARCHAR(20)  NOT NULL CHECK (plan IN ('basic', 'pro', 'enterprise')),
    status          VARCHAR(20)  NOT NULL DEFAULT 'active'
                        CHECK (status IN ('active', 'paused', 'canceled', 'expired')),
    started_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    expires_at      TIMESTAMPTZ  NOT NULL,
    trial_ends_at   TIMESTAMPTZ,
    auto_renew      BOOLEAN      NOT NULL DEFAULT FALSE,
    payment_method_id BIGINT,
    created_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_subscriptions_user_id ON subscriptions (user_id);
CREATE INDEX idx_subscriptions_status  ON subscriptions (status);
CREATE INDEX idx_subscriptions_expires ON subscriptions (expires_at);
```

#### payments（支付表）

```sql
CREATE TABLE payments (
    id                  BIGSERIAL PRIMARY KEY,
    subscription_id     BIGINT        NOT NULL,
    amount              INT           NOT NULL,          -- 单位：分（避免浮点精度问题）
    currency            VARCHAR(3)    NOT NULL DEFAULT 'CNY',
    provider            VARCHAR(20)   NOT NULL
                            CHECK (provider IN ('stripe', 'alipay', 'wechat')),
    provider_payment_id VARCHAR(128),
    status              VARCHAR(20)   NOT NULL DEFAULT 'pending'
                            CHECK (status IN ('pending', 'paid', 'failed', 'refunded')),
    paid_at             TIMESTAMPTZ,
    created_at          TIMESTAMPTZ   NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_payments_subscription_id ON payments (subscription_id);
CREATE INDEX idx_payments_provider_id     ON payments (provider_payment_id);
```

#### invoices（发票表）

```sql
CREATE TABLE invoices (
    id              BIGSERIAL PRIMARY KEY,
    user_id         BIGINT       NOT NULL,
    payment_id      BIGINT       NOT NULL,
    invoice_number  VARCHAR(64)  NOT NULL UNIQUE,
    pdf_url         VARCHAR(512),
    issued_at       TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    created_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_invoices_user_id    ON invoices (user_id);
CREATE INDEX idx_invoices_payment_id ON invoices (payment_id);
```

#### coupons（优惠券表）

```sql
CREATE TABLE coupons (
    id              BIGSERIAL PRIMARY KEY,
    code            VARCHAR(64)   NOT NULL UNIQUE,
    discount_type   VARCHAR(10)   NOT NULL CHECK (discount_type IN ('percentage', 'fixed')),
    discount_value  INT           NOT NULL,              -- 百分比(1-100) 或 固定金额(分)
    max_uses        INT           NOT NULL DEFAULT 0,    -- 0 表示不限
    current_uses    INT           NOT NULL DEFAULT 0,
    expires_at      TIMESTAMPTZ   NOT NULL,
    created_at      TIMESTAMPTZ   NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_coupons_code ON coupons (code);
```

### 2.2 ER 关系

```
users 1 ──── N subscriptions 1 ──── N payments 1 ──── N invoices
                                      │
                                      └── coupons (独立，支付时校验核销)
```

## 3. API 端点

### 3.1 订阅管理

#### POST /api/v1/subscriptions

创建订阅（新用户或到期后重新订阅）。

```json
// Request
{
    "plan": "pro",
    "payment_method_id": 1,
    "coupon_code": "WELCOME2026",      // 可选
    "auto_renew": true
}

// Response 201
{
    "subscription_id": 42,
    "plan": "pro",
    "status": "active",
    "started_at": "2026-07-02T00:00:00Z",
    "expires_at": "2027-07-02T00:00:00Z",
    "trial_ends_at": "2026-07-09T00:00:00Z",
    "auto_renew": true
}
```

#### GET /api/v1/subscriptions/me

获取当前用户订阅状态。

```json
// Response 200
{
    "subscription_id": 42,
    "plan": "pro",
    "status": "active",
    "started_at": "2026-07-02T00:00:00Z",
    "expires_at": "2027-07-02T00:00:00Z",
    "trial_ends_at": "2026-07-09T00:00:00Z",
    "auto_renew": true,
    "days_remaining": 365,
    "trial_active": true,
    "trial_days_remaining": 7
}
```

#### PATCH /api/v1/subscriptions/me

升级、降级或取消订阅。

```json
// Request — 升级
{
    "action": "upgrade",
    "target_plan": "enterprise"
}

// Request — 取消自动续费
{
    "action": "cancel_auto_renew"
}

// Request — 立即取消（不退费，权限保留至到期日）
{
    "action": "cancel"
}

// Response 200
{
    "subscription_id": 42,
    "plan": "enterprise",           // 升级后 reflect
    "status": "active",
    "started_at": "2026-07-02T00:00:00Z",
    "expires_at": "2026-10-02T00:00:00Z",  // 按比例折算
    "auto_renew": true,
    "prorated_amount": 1725         // 应补差价（单位：分）
}
```

**升级/降级按比例折算逻辑**：
- 升级：`新价格 / 365 * 剩余天数 - 旧价格 / 365 * 剩余天数`，结果 ≥ 0 则立即扣款，< 0 则存入余额（仅降级场景）。
- 降级：差价存入账户余额，下次续费时抵扣。

### 3.2 支付

#### POST /api/v1/payments/checkout

发起支付，返回支付链接或二维码。

```json
// Request
{
    "subscription_id": 42,
    "amount": 19900,                // 单位：分
    "provider": "wechat",
    "return_url": "https://sci-agent.com/payment/result"
}

// Response 200 — 微信/支付宝
{
    "payment_id": 1024,
    "provider": "wechat",
    "qr_code_url": "https://api.sci-agent.com/payments/1024/qrcode",
    "expires_at": "2026-07-02T00:30:00Z"
}

// Response 200 — Stripe
{
    "payment_id": 1025,
    "provider": "stripe",
    "checkout_url": "https://checkout.stripe.com/c/pay/cs_test_xxx",
    "expires_at": "2026-07-02T01:00:00Z"
}
```

#### POST /api/v1/payments/webhook

支付回调（由 Stripe / 支付宝 / 微信支付服务端调用）。

```json
// Request — Stripe 示例
{
    "type": "checkout.session.completed",
    "data": {
        "object": {
            "id": "cs_test_xxx",
            "metadata": {
                "payment_id": "1025"
            },
            "amount_total": 19900,
            "currency": "cny",
            "payment_status": "paid"
        }
    }
}
```

**幂等处理逻辑**：

```python
# 伪代码
def handle_webhook(provider, provider_payment_id, status):
    payment = Payment.query.filter_by(
        provider=provider,
        provider_payment_id=provider_payment_id
    ).first()

    if not payment:
        log.warning(f"Unknown payment: {provider_payment_id}")
        return 404

    if payment.status == status:
        return 200  # 已处理，幂等返回

    with db.transaction():
        payment.status = status
        if status == "paid":
            payment.paid_at = utcnow()
            subscription = Subscription.query.get(payment.subscription_id)
            subscription.status = "active"
            subscription.expires_at = utcnow() + timedelta(days=365)
            # 生成发票
            issue_invoice(payment)
```

## 4. 支付集成

### 4.1 架构总览

```
┌─────────────────────────────────────────────────────┐
│                    SCI-Agent API                     │
│                                                      │
│   POST /checkout ──► PaymentService ──► ProviderSDK  │
│   POST /webhook  ◄── (回调)                           │
│                                                      │
├─────────────────────────────────────────────────────┤
│              PaymentService (统一抽象)                │
│                                                      │
│   ┌──────────┐  ┌──────────┐  ┌──────────┐          │
│   │  Stripe  │  │  Alipay  │  │  WeChat  │          │
│   │  Adapter │  │  Adapter │  │  Adapter │          │
│   └────┬─────┘  └────┬─────┘  └────┬─────┘          │
│        │              │              │                │
└────────┼──────────────┼──────────────┼────────────────┘
         │              │              │
    ┌────▼────┐   ┌────▼────┐   ┌────▼────┐
    │ Stripe  │   │ Alipay  │   │ WeChat  │
    │  API    │   │  SDK    │   │ Pay SDK │
    └─────────┘   └─────────┘   └─────────┘
```

### 4.2 渠道策略

| 渠道 | 覆盖区域 | 集成方式 | 手续费 |
|---|---|---|---|
| **Stripe** | 国际（信用卡） | Stripe Python SDK + Webhook | ~2.9% + $0.30 |
| **支付宝** | 国内 | alipay-sdk-python / 当面付 | ~0.6% |
| **微信支付** | 国内 | wechatpy / JSAPI / Native | ~0.6% |

### 4.3 国内支付方案说明

支付宝和微信支付可通过以下两种方式接入：

- **方案 A（推荐）**：通过 Stripe 聚合接入。Stripe 已在中国支持 Alipay 和 WeChat Pay，统一 SDK、统一 Webhook，减少维护成本。
- **方案 B**：直接对接支付宝 SDK 和微信支付 SDK，需分别处理签名、回调、证书轮换，适合对费率敏感的 B 端部署。

### 4.4 Webhook 幂等处理

```
┌──────────────────┬──────────────────────────────────────┐
│ 策略             │ 说明                                  │
├──────────────────┼──────────────────────────────────────┤
│ provider_payment_id 去重 │ 回调先查是否已存在同一 provider_payment_id │
│ 状态机校验        │ pending→paid→refunded，不允许跳过状态  │
│ DB 事务          │ payment 和 subscription 同事务更新     │
│ 回调签名验证      │ Stripe: webhook signing secret       │
│                  │ Alipay: RSA 签名验签                   │
│                  │ WeChat: APIv3 签名                     │
│ 回调响应码        │ 处理成功返回 200，否则返回非 200       │
│                  │ 支付网关会重试失败回调（通常 3~5 次）   │
└──────────────────┴──────────────────────────────────────┘
```

## 5. 权限控制

### 5.1 中间件 verify_subscription

```python
from functools import wraps
from flask import request, jsonify, g

def verify_subscription(f):
    """验证当前用户订阅是否有效"""
    @wraps(f)
    def decorated(*args, **kwargs):
        user = g.current_user
        sub = Subscription.query.filter_by(
            user_id=user.id,
            status="active"
        ).order_by(Subscription.expires_at.desc()).first()

        if not sub or sub.expires_at < utcnow():
            return jsonify({
                "error": "subscription_required",
                "message": "此功能需要有效订阅，请前往升级。"
            }), 402

        g.subscription = sub
        return f(*args, **kwargs)
    return decorated

def require_plan(plan):
    """功能门控：要求指定 plan 及以上"""
    def decorator(f):
        @wraps(f)
        def decorated(*args, **kwargs):
            sub = g.subscription
            plan_map = {"basic": 0, "pro": 1, "enterprise": 2}
            if plan_map.get(sub.plan, -1) < plan_map[plan]:
                return jsonify({
                    "error": "upgrade_required",
                    "message": f"此功能需要 {plan} 及以上订阅。"
                }), 403
            return f(*args, **kwargs)
        return decorated
    return decorator
```

### 5.2 使用示例

```python
# 需要有效订阅
@router.post("/api/v1/projects")
@verify_subscription
def create_project():
    ...

# 需要 Pro 及以上
@router.get("/api/v1/advanced-filters")
@verify_subscription
@require_plan("pro")
def get_advanced_filters():
    ...
```

| 功能 | Basic | Pro | Enterprise |
|---|---|---|---|

| 高级检索过滤器 | ❌ | ✅ | ✅ |

## 6. 免费试用

### 6.1 试用策略

| 项 | 值 |
|---|---|
| 试用时长 | 7 天 |
| 适用用户 | 首次注册（`subscriptions` 表无历史记录） |
| 试用权益 | 等同于 Pro 计划 |
| 自动续费 | 试用期间 `auto_renew=true` 则到期自动扣款 |

### 6.2 trial_ends_at 字段

已在 `subscriptions` 表中定义（见 §2.1）。注册时自动填充：

```python
def create_trial_subscription(user_id: int) -> Subscription:
    """首次注册创建试用订阅"""
    now = utcnow()
    sub = Subscription(
        user_id=user_id,
        plan="pro",
        status="active",
        started_at=now,
        expires_at=now + timedelta(days=7),
        trial_ends_at=now + timedelta(days=7),
        auto_renew=False
    )
    db.session.add(sub)
    db.session.commit()
    return sub
```

### 6.3 到期提醒

通过定时任务（Celery / APScheduler）扫描：

```python
# 每日凌晨 8:00 执行
def send_trial_reminders():
    """到期前 3 天发送邮件提醒"""
    target_date = utcnow() + timedelta(days=3)
    subs = Subscription.query.filter(
        Subscription.trial_ends_at >= target_date,
        Subscription.trial_ends_at < target_date + timedelta(days=1),
        Subscription.status == "active"
    ).all()

    for sub in subs:
        user = User.query.get(sub.user_id)
        send_email(
            to=user.email,
            subject="SCI-Agent 试用即将到期",
            template="emails/trial_expiring.html",
            context={
                "username": user.username,
                "trial_ends_at": sub.trial_ends_at,
                "days_left": 3,
            }
        )
```
