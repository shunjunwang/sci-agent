## 1. 用户引导体系（Onboarding）

### 1.1 首次使用引导

新用户首次登录后，进入 **3 步引导流程**，整体耗时控制在 60 秒以内。

| 步骤 | 页面 | 用户操作 | 引导文案（要点） |
|------|------|----------|------------------|
| Step 1 | 身份选择弹窗 | 选择角色：`研究生` / `科研人员` / `企业研发` | "选择你的身份，我们将为你定制专属工作台" |
| Step 2 | 文献导入引导 | 拖拽 BibTeX 文件 或 关联 ORCID / Google Scholar | "导入现有文献库，或直接跳过，稍后随时补充" |
| Step 3 | 任务引导卡片 | 点击卡片完成首个任务（如"搜索一篇论文并加入文献库"） | "恭喜！你已完成首次任务。探索更多功能吧" |

**交互设计细则**：

- 每一步可点击「跳过」，跳过后在对应模块保留"完成引导"的快捷入口。
- 引导流程结束后，展示 3 秒的完成动画并自然过渡到主界面。
- 引导进度以顶部进度条 + 步骤序号 (1/3) 展示。

### 1.3 功能发现：自然语言指令搜索

顶部全局搜索框支持**自然语言指令**，自动识别用户意图并跳转到对应功能模块。

**技术实现**：前端对输入文本做意图分类（轻量规则 + LLM 兜底），映射到路由跳转。

| 用户输入示例 | 识别意图 | 路由跳转 |
|-------------|---------|---------|
| "帮我写论文大纲" | AI 写作 - 大纲生成 | `/write?action=outline` |
| "搜索深度学习论文" | 文献检索 | `/library/search?q=深度学习` |
| "创建关于 Transformer 的笔记" | 知识库 - 新建笔记 | `/knowledge/new?tag=Transformer` |
| "运行一个 Python 脚本" | 仿真沙箱 | `/sandbox` |
| "导出参考文献" | 文献库 - 导出 | `/library?action=export` |

**搜索框交互**：

- 输入时实时展示意图分类结果（图标 + 模块名），回车或点击确认跳转。
- 未匹配到已知意图时，降级为全文搜索（文献库内容）。
- 搜索框 placeholder 动态轮播："试试说「帮我写论文大纲」"、"搜索论文、笔记或命令…"

## 2. 国际化（i18n）架构

### 2.1 技术选型

| 端 | 技术方案 | 说明 |
|----|---------|------|
| 前端 (Next.js) | `next-intl` | 支持 App Router、SSR/SSG 下的国际化路由、类型安全 |
| 后端 (Python) | `gettext` + `Babel` | 成熟的 GNU gettext 体系，`.po` / `.mo` 文件管理 |
| 格式化工具 | `Intl` API | 日期、时间、数字、货币的本地化格式化，统一走浏览器内置 `Intl` |

#### 2.2.4 日期/时间/数字格式化

所有格式化统一使用 `Intl` API，禁止硬编码格式串。

```tsx
// ✅ 正确：走 Intl，自动适配语言环境
const formattedDate = new Intl.DateTimeFormat(locale, {
  year: 'numeric',
  month: 'long',
  day: 'numeric',
}).format(new Date());

const formattedNumber = new Intl.NumberFormat(locale).format(12345);

// ❌ 错误：硬编码中文格式
// const date = `${year}年${month}月${day}日`;
```

### 2.3 后端实现

#### 2.3.1 语言文件结构

```
backend/
├── locales/
│   ├── zh_CN/
│   │   └── LC_MESSAGES/
│   │       ├── messages.po      # 翻译源文件
│   │       └── messages.mo      # 编译后的二进制文件
│   └── en_US/
│       └── LC_MESSAGES/
│           ├── messages.po
│           └── messages.mo
└── babel.cfg                    # Babel 提取配置
```

#### 2.3.2 Python 代码中使用

```python
import gettext
import locale

# 根据用户偏好设置语言
def setup_i18n(preferred_language: str = "zh_CN"):
    locale_dir = Path(__file__).parent / "locales"
    lang = gettext.translation(
        "messages",
        localedir=str(locale_dir),
        languages=[preferred_language],
        fallback=True,
    )
    _ = lang.gettext
    return _

_ = setup_i18n()

# 业务代码中所有字符串走 _() 函数
raise ValueError(_("Invalid DOI format: {doi}").format(doi=user_input))
logger.info(_("Literature import completed: {count} records").format(count=len(records)))
```

#### 2.3.3 messages.po 示例（zh_CN）

```po
# Chinese translations for sci-agent backend
msgid ""
msgstr ""
"Language: zh_CN\n"
"Content-Type: text/plain; charset=UTF-8\n"

msgid "Invalid DOI format: {doi}"
msgstr "无效的 DOI 格式：{doi}"

msgid "Literature import completed: {count} records"
msgstr "文献导入完成：共 {count} 条记录"
```

### 2.4 语言切换策略

| 层级 | 策略 |
|------|------|
| 前端路由 | Next.js 中间件从 Cookie / Accept-Language 读取，设置 locale |
| 后端 API | 请求头 `Accept-Language` → 注入当前请求的 gettext 实例 |
| 数据库 | `users` 表增加 `preferred_language VARCHAR(10) DEFAULT 'zh-CN'` |
| 默认语言 | zh-CN（中文）；架构已预留英文扩展，所有字符串走翻译函数 |

初期只维护 `zh-CN`，`en-US` 文件保留 key 结构，value 用中文占位或英文机翻标注 `[TODO: translate]`，后期由人工覆盖翻译。

**渲染方案**：

- 帮助中心路由：`/help/[...slug]` → 映射到 `public/help/` 下的对应文件。
- 左侧固定导航树，右侧渲染 Markdown 内容。

示例路由映射：

```
/help                          → index.md
/help/quickstart/5min-intro    → quickstart/5min-intro.md
/help/faq                      → faq.md
```

### 3.4 文档更新机制

| 机制 | 说明 |
|------|------|
| **版本绑定** | 每个版本文档存放在 `help/v{version}/` 下，发布时通过软链接或构建时复制到 `public/help/` |
| **CI 检查** | PR 合入时自动运行脚本，检查以下内容： |
| | ① `faq.md` 是否与 `CHANGELOG.md` 的新功能/修复对应更新 |
| | ② 帮助文档内引用的 UI 截图路径是否存在 |
| | ③ Markdown 内链是否有效（不产生 404 死链） |
| **翻译同步** | 英文版帮助文档有单独的 `help/en/` 目录；CI 检查中英文文档的章节结构是否一致 |
| **定期审查** | 每个版本迭代结束时，由 PM + 技术写作审查帮助文档完整性 |

**CI 检查脚本示例**：

```yaml
# .github/workflows/help-docs-check.yml
name: Help Docs Check
on:
  pull_request:
    paths:
      - 'docs/**'
      - 'frontend/public/help/**'

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Check markdown links
        run: |
          npx markdown-link-check frontend/public/help/**/*.md
      - name: Check image references
        run: |
          python scripts/check_help_images.py
```
