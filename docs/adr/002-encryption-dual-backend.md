# ADR-002: 加密算法 — Fernet + AES-256-GCM 双后端

- **状态**：已决定
- **日期**：2026-07-04
- **决策者**：人 + AI
- **关联 ISSUE**：P1-17

## 上下文

SPEC 要求使用 AES-256-GCM 加密，但实际实现使用了 Fernet（AES-128-CBC）。两者安全级别和性能差异显著。直接替换会破坏已有加密数据。

## 选项

| 选项 | 优点 | 缺点 |
|------|------|------|
| A: 纯 Fernet | 实现简单、已有数据兼容 | 不满足 SPEC 安全要求（128 位 vs 256 位） |
| B: 纯 AES-256-GCM | 满足 SPEC、更强安全性 | 破坏已有加密数据、需强制迁移 |
| C: 双后端 | 兼容已有数据 + 新数据用强加密 | 维护两套代码 |

## 决策

选择 **C：双后端设计**。

- 已有数据保持 Fernet 解密
- 新加密数据统一使用 AES-256-GCM
- 提供迁移工具将旧数据逐步转换（非强制）

## 后果

- `core/encryption.py` 维护两套后端，增加约 40 行代码
- 需要 `ENCRYPTION_MODE` 环境变量区分模式
- 长期目标：迁移比例达 95% 后可废弃 Fernet 后端
